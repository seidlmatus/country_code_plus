# Contributing

Thanks for contributing to `country_codes_plus`.

## Development Setup
1. Install Flutter (stable channel).
2. Run `flutter pub get` in the project root.
3. Run checks before creating a PR:
   - `dart analyze`
   - `flutter test`

## Pull Request Guidelines
1. Keep changes focused and small.
2. Add or update tests for behavioral changes.
3. Update docs when public behavior changes.
4. For release-impacting changes, update:
   - `CHANGELOG.md`
   - `RELEASE_NOTES.md`

## Commit Messages
- Use clear, imperative messages (example: `Fix Android plugin package registration`).

## Reporting Issues
- Include Flutter version, platform, expected behavior, and actual behavior.
- If possible, provide a minimal reproducible example.
